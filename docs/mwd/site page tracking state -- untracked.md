[[site page tracking state]]

If [[constrained graph expansion]] leads to a [[source page]] becoming a [[site page]] , then that site page will be [[site page tracking state -- untracked]] at first.

In the [[app component -- site page views]], the [[publisher]] will need to make a decision about whether to include that site page in the [[tracked working graph]] or not.

You can't do a [[publish mode -- clean]] without all graph pages being registered.  Also see [[publish mode -- dirty]] for other reasons it can be dirty.

Untracked pages show in a special [[app component -- warning - untracked site pages]]
