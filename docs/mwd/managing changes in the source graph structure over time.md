Managing [[source graph change]]

Unlike standard [[publishing modality -- blog]], the [[publishing modality -- meadow site]]s are backed by [[source graph]]s that change over time.  Part of our [[meadow change management]] challenge is related to dealing with these [[source graph]] structural changes.

When the source graph changes you can end up with [[orphaned tracked site page]]s, or a bunch of newly [[site page tracking state -- untracked]] pages within the [[working graph controlled zone]].

This can be especially problematic at scale, and needs to be addressed by [[multi site management]].  But you don't necessarily care about all of your sites equally.  Maybe you make some sites as one-offs, which is why we've added [[site state -- archived]] as a type (shunted off into another tab).
