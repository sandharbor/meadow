^ multi [[site]] management

---

Part of a broader theme of [[meadow change management]]

Multi site management refers to the capability to manage multiple [[site]]s effectively. This concept is closely tied to [[filter page selector scope -- global]] and enables users to:

- Apply [[filter]]s across multiple sites
- Manage sensitive content across publications
- [[managing changes in the source graph structure over time]]
	- perhaps you'd be fine with just adding some specific, [[dim - sensitivity -- low]] pages to all the sites and re-publishing all of them
	- or you know you deleted some page and you'd be happy to remove it from being [[site page tracking state -- tracked]] in all the graphs.  Or maybe some visualization would help you know whether it would be ok to remove.

Maybe we allow you indicate that you are happy with a specific version of a site, so we don't keep telling you that there are changes within the [[working graph controlled zone]].  I think this could be [[site state -- archived]]
