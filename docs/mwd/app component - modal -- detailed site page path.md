[[app component]] - [[app component - modal]]

To serve our [[design motivation -- understandability]], we provide a more detailed view of [[app component -- site page path]], where we show the details of the path from the [[site page type -- initial]] to the selected site page.

When there is any _link not tracked_, we show a _link not tracked_.

In the [[site page path]] to a _link not tracked_, without the details, it was hard to tell _why_ a page was included.  Was it because of all [[outlink]]s, or is there one or more [[inlink]]s to another page in the path?  Since the application of [[outlink]]s or [[inlink]]s is inherited, and the depth of each is independent, like [[site page config -- inlinksDepth]], this detailed information is important to know... especially understanding why _link not tracked_s are included.
