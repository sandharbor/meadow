One driving goal is [[design motivation -- great change management]].

The [[app component - modal -- site preview -- tab changes|changes tab]] only shows the changes to the result of [[html generation]], not to:
* the [[meadow config directory path - tracked markdown - HOME slash .config slash meadow slash sites slash the site slash raw slash tracked_page_content|tracked markdown]], which *can* be equally interesting, especially if you are creating a [[meadow markdown export type -- published to site as zip file|published markdown zip file]]
* the various [[scope -- site]] [[site config]] files, including those used in [[app component -- sidebar -- customize preview|customize sidebar]].  This feels like a gap.  Yes, you can see those changes if you go to [[meadow home directory default path - HOME slash Library slash Application Support slash Meadow slash MeadowHome|the meadowHome directory]] and look at it in [[project - git|git]], but the whole point of this app is to shield you from needing to do that.
* Also, what if the user made [[scope -- global]] changes while using the [[app component -- sidebar -- customize preview|customize sidebar]]?  Perhaps _those_ should show up in the diff, as well.

All that said, [[maybe the customize sidebar should have its own change management]]
