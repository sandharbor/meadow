By Alistair Cockburn. Classic book on structuring use cases for software requirements.

Offers a disciplined approach to describing how users interact with a system — structured templates, levels of detail, and clear success/failure scenarios.

Relevant to the question of how to best describe software systems. Use cases are one structured approach that sits between free-form prose and [[type-safe specification - 2026-02]] — more disciplined than markdown, but not machine-checkable.
